ecdsaCanonicality

Determines the canonicality of a signature. 

boost::optional< ECDSACanonicality >
ecdsaCanonicality(
    Slice const& sig);

A canonical signature is in its most reduced form. For example the R and S components do not contain additional leading zeroes. However, even in canonical form, (R,S) and (R,G-S) are both valid signatures for message M.

Therefore, to prevent malleability attacks we define a fully canonical signature as one where: R < G - S

where G is the curve order.

This routine returns boost::none if the format of the signature is invalid (for example, the points are encoded incorrectly).

Return Value

boost::none if the signature fails validity checks.

Remarks

Only the format of the signature is checked, no verification cryptography is performed.
